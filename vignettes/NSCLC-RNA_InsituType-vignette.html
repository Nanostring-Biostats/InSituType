Overview

This vignette demonstrates the use of the Insitutype algorithm to cluster CosMx RNA data. We’ll analyze a small subset of the non-small cell lung cancer (NSCLC) 1000-plex public data release.

For a deep-dive in the nuances of running Insitutype and QC-ing its results, see the FAQS.md file.

  1. Optionally, a vector of each cell’s expected background counts (per gene). If not provided, Insitutype will estimate this value from the negative controls.

  2. For supervised or semi-supervised mode, a reference profile matrix giving the expected expression profile of each cell type, with genes in rows and cell types in columns. The reference matrix must be linear-scale, not log-scale. Insitutype will automatically align the gene in this matrix to your expression data.

For supervised cell typing, reference matrices should contain every cell type present in the tissue. If you suspect the tissue has cell types outside your reference matrix, use Insitutype’s semi-supervised clustering capability to detect new clusters.

Incorporating additional data types

Single-cell spatial transcriptomics datasets offer data types beyond simple gene expression, from single-cell images to spatial contexts. Insitutype digests this information in the form of “cohorts” of cells with similar data from the alternative data types. Then, when assigning cell types, Insitutype will considers the frequency of each cell type in each cohort.

For example, if you have a cohort of mainly CD45- cells, Insitutype will see that immune cells are very uncommon within this cohort, and it will give cells in this cohort lower probabilities of falling in immune cell clusters. (Insitutype doe not know about CD45 staining; it just considers the frequency of each cluster in each cohort.)

Cohort definition is left to the user, and many approaches are reasonable. For example, you might:

  • Define cohort by gating cells on their immunofluorescence stains, e.g. PanCK+/- or CD45+/-
  • Cluster cells on their continuous immunofluorescence values
  • Use an autoencoder to extract features from cells images, then cluster on those features.
  • Use the results of spatial clustering / “niches” to define cohorts
  • Cluster on information from multiple data types together, such as the average expression profile of cell neighbors AND immunofluorescence.

Cohorting is a low-risk undertaking, so it is recommended whenever possible. A poorly-defined cohort will simply be uninformative; it will not send the clustering algorithm astray. The more cohorts you can define, and the higher-quality those cohorts, the more information Insitutype will be able to gleam from the alternative data types.

One warning: do not define cohorts based on variables you want to study. For example, if you wish to study how cell populations shift across spatial context, then spatial context must not be allowed to inform cell typing. To do so would introduce circularity.

In the below example, we will define cohorts using the “fastCohorting” function. This function is provided as a convenient way to derive cohorts from diverse data types; it makes no claim to being optimal.

# simulate immunofluorescence data: 
immunofluordata <- matrix(rpois(n = nrow(counts) * 4, lambda = 100), nrow(counts))
# perform automatic cohorting:
cohort <- fastCohorting(immunofluordata, gaussian_transform = TRUE) 
# ("Gaussian_transform = TRUE" maps variables to Gaussians in order to 
#  place dramatically different variables on the same scale.)
table(cohort)
#> cohort
#>    1    2    3 
#> 1236  631  331

Supervised cell typing:

The below command performs supervised cell typing The user needs to provide reference_profiles and set n_clusts=0. For supervised cell typing, there is usually no need to update the reference profile, though if your initial results are unsatisfying then setting update_reference_profiles = FALSE can help.

sup <- insitutype(
  x = counts,
  neg = negmean,
  assay_type = "RNA", 
  n_clusts=0, 
  anchors = NULL, 
  reference_profiles = ioprofiles, 
  cohort = cohort,
  update_reference_profiles = FALSE
)

Semi-supervised clustering:

The below command performs semi-supervised clustering. For semi-supervised clustering, it is usually necessary to update your reference profiles to better fit CosMx data. Insitutype uses “anchor” cells with high-confidence cell types to perform this calibration. You can specify anchors yourself, or, as below, you can let the algorithm choose them automatically. For semi-supervised clustering, we specify a non-zero n_clusts value and supply a reference profiles matrix.

semi_sup <- insitutype(
  x = counts,
  neg = negmean,
  assay_type = "RNA", 
  n_clusts = 3, 
  anchors = NULL, 
  reference_profiles = ioprofiles, #[names(sig_mat)[-c(1:2)],],
  cohort = cohort,
  update_reference_profiles = TRUE)

Insitutype calculates the probability of each cell belonging to each cluster. This information lets us understand how clusters relate to each other. The flightpath_plot function arrays cells according to their cluster probabilities:

# make the flightpath plot
fp <- flightpath_plot(flightpath_result = NULL, insitutype_result = unsup, col = cols[unsup$clust])
#> Saving flightpath_plot to: ./NBClust-Plots/2024-02-21_10-32-08-PST-flightpath_plot.png
class(fp)
#> [1] "gg"     "ggplot"
print(fp)

In the above plot, a cell atop a cluster centroid has near-certain cell type, while cells halfway between two centroids have 50% posterior probability of belonging to each of those two clusters. The number atop each centroid gives the average posterior probability of cells in the cluster. Expect to see this value above 0.9 for clustering major cell types, and above 0.75 when clustering apart closely-related cell types. Clusters with low average posterior probabilities are unreliable. You can either interpret them with caution or you can dissolve them and reassign their cells to other clusters. (The next section covers this operation.)

Updating clustering results

After close review, you might want to update your clustering results.

The function refineClusters can modify your results in the following ways:

  • Merge closely-related or frequently-confused clusters
  • Delete clusters (Sometimes a “catch-all” cluster arises that is frequently confused with multiple other clusters. It is best to delete these clusters and let their cells get reassigned to meaningful clusters.)
  • Sub-cluster large or diverse clusters

Here we use the refineClusters function to perform all these operations. We also use its “merges” argument to rename cluster c.

# refine the clusters:
newclusts <- refineClusters(logliks = unsup$logliks, 
                            merges = c("g" = "newcluster", "e" = "newcluster", "c" = "c_renamed"), 
                            to_delete = "a",
                            subcluster = c("f" = 2),
                            counts = counts,

Coloring cell types

The colorCellTypes function attempts to choose good colors for your cell types. Some of its options use the frequency of cell types in this decision. Here are 3 versions:
